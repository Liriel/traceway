# Node.js (OpenTelemetry)

Instrument any Node.js backend — Express, Fastify, Koa, Hono, NestJS, or plain HTTP — with OpenTelemetry and export traces and metrics to Traceway.

## Installation

```bash
npm install @opentelemetry/sdk-node \
  @opentelemetry/auto-instrumentations-node \
  @opentelemetry/exporter-trace-otlp-http \
  @opentelemetry/exporter-metrics-otlp-http
```

## Setup

Create an `instrumentation.ts` (or `.js`) file at the root of your project. This file must be loaded **before** your application code.

```typescript
import { NodeSDK } from "@opentelemetry/sdk-node";
import { OTLPTraceExporter } from "@opentelemetry/exporter-trace-otlp-http";
import { OTLPMetricExporter } from "@opentelemetry/exporter-metrics-otlp-http";
import { PeriodicExportingMetricReader } from "@opentelemetry/sdk-metrics";
import { getNodeAutoInstrumentations } from "@opentelemetry/auto-instrumentations-node";
import { Resource } from "@opentelemetry/resources";

const sdk = new NodeSDK({
  resource: new Resource({
    "service.name": "my-service",
    "service.version": "1.0.0",
  }),

  traceExporter: new OTLPTraceExporter({
    url: "https://your-traceway-instance.com/api/otel/v1/traces",
    headers: { Authorization: "Bearer your-project-token" },
  }),

  metricReader: new PeriodicExportingMetricReader({
    exporter: new OTLPMetricExporter({
      url: "https://your-traceway-instance.com/api/otel/v1/metrics",
      headers: { Authorization: "Bearer your-project-token" },
    }),
    exportIntervalMillis: 30_000,
  }),

  instrumentations: [getNodeAutoInstrumentations()],
});

sdk.start();
```

## Running Your App

Load the instrumentation file before everything else using Node's `--require` (CommonJS) or `--import` (ESM) flag:

```bash
# CommonJS
node --require ./instrumentation.js server.js

# ESM / TypeScript (with tsx or ts-node)
node --import ./instrumentation.ts server.ts
```

Or set it via environment variable:

```bash
export NODE_OPTIONS="--require ./instrumentation.js"
node server.js
```

## Environment Variable Alternative

Instead of hardcoding the endpoint and token, you can use standard OTel environment variables:

```bash
export OTEL_SERVICE_NAME="my-service"
export OTEL_EXPORTER_OTLP_ENDPOINT="https://your-traceway-instance.com/api/otel"
export OTEL_EXPORTER_OTLP_HEADERS="Authorization=Bearer your-project-token"
```

With these set, the SDK picks them up automatically — your `instrumentation.ts` can omit the exporter URLs and headers.

## What Gets Auto-Instrumented

The `@opentelemetry/auto-instrumentations-node` package automatically instruments:

| Library | What's Captured |
|---------|-----------------|
| `http` / `https` | Incoming and outgoing HTTP requests |
| `express` | Route-level spans with `http.route` |
| `fastify` | Route-level spans |
| `koa` | Middleware and route spans |
| `hono` | HTTP request spans (via `http` instrumentation) |
| `pg` / `mysql2` / `mongodb` | Database query spans |
| `redis` / `ioredis` | Cache operation spans |
| `grpc` | gRPC call spans |
| `fs` | File system operation spans |

Root SERVER spans become **Endpoints** in Traceway, child spans become **Spans**, and exception events become **Issues**. See [Traces](/client/otel/traces) for the full mapping.

## Framework Quick Start

### Express

```typescript
import express from "express";

const app = express();

app.get("/api/users", (req, res) => {
  res.json({ users: [] });
});

app.listen(3000);
```

No extra code needed — auto-instrumentation captures all requests, routes, and database calls.

### Fastify

```typescript
import Fastify from "fastify";

const app = Fastify();

app.get("/api/users", async () => {
  return { users: [] };
});

app.listen({ port: 3000 });
```

### Hono

See the dedicated [Hono guide](/client/hono) for multi-runtime setup.

```typescript
import { serve } from "@hono/node-server";
import { Hono } from "hono";

const app = new Hono();

app.get("/api/users", (c) => c.json({ users: [] }));

serve({ fetch: app.fetch, port: 3000 });
```

## Test Your Integration

Add a route that throws an error:

```typescript
app.get("/test-error", () => {
  throw new Error("Test error from Traceway integration");
});
```

Visit `/test-error` and check your Traceway dashboard — the error should appear under **Issues** with a full stack trace.
